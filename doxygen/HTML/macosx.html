Updating gcc

At the time of writing this guide, the version of gcc that ships with Xcode is unable to compile MODEL. However, you can easily install a more recent version of gcc in your Mac OsX using MacPorts. After installing MacPorts, run the following in Terminal: 

sudo port install gcc8

This installs gcc6 in your system. Now make sure to select gcc6 as the default gcc compiler. You can select a particular version of gcc by first installing gcc_select, again via MacPorts: 

sudo port install gcc_select

Then, list the available gcc versions in your system: 

port select --list gcc

And finally select the most current version, say mp-gcc8: 

sudo port select --set gcc mp-gcc8

For DDmpi, also install the following 

sudo port install openmpi-gcc6
sudo port install mpi_select

and select the current mpi version 

port select --list mpi
sudo port select --set mpi openmpi-gcc6-fortran
sudo port install openmpi

Compiling DDvtk

In order to compile DDvtk we need cmake and vtk from MacPorts 

sudo port install cmake
sudo port install vtk

Unfortunately, vtk is only compatible with the system g++/gcc (clang). Therefore, when invoking cmake, we need to specify the system compiler: 

cd tools/DDvtk 
cmake . -DCMAKE_CXX_COMPILER=/usr/bin/g++ -DCMAKE_CC_COMPILER=/usr/bin/gcc
make

Make sure to complete these steps before installing open-MPI.

Porting other Unix/Linux packages

MacPorts can also be used to port the following packages to Mac OsX:

  • ImageMagick
  • FFmpeg
